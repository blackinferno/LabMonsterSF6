Lab Monster SF6 - Usage Guide

Definitive combo management tool for combo authoring, importing/exporting, safe data management, and frame-data reference.

Frame Data

Frame Data Table

  • A copy of the latest frame data table on the official website. Same data, with key values color-coded.

Version Switching

  • You can switch to older frame data tables (as far back as 2025.11.13 update).

Version Comparison

  • You can compare the frame data table with other versions.
  • Changed cells will be highlighted.
  • You can see exact changes in a tooltip by hovering over a highlighted cell.

Combo List

Combo Authoring

  • Per-character and per-control-mode combo table (Classic / Modern).
  • Comprehensive fields for conditions, damage variants, resources, carry, and post-combo state.
  • Multiline notes field for setup details and other memos.

Editing / Maintenance Tools

  • Row operations: create, duplicate, delete, and strict dedupe.
  • Sortable headers/subheaders for quick data review.
  • Auto-generated button icons from command notation.

View / Workflow Controls

  • Preset column views: FULL, BASIC, SIMPLE, and CUSTOM.
  • Toggle visibility of Frame Meter / Buttons / Notes rows.
  • Collapsible lower input section for larger table workspace.

Search / Filter

  • Quick search for broad text/value lookup.
  • Advanced Search for field-scoped and range-based filtering.

Input / Device Support

  • Manual command entry, quick-input icons, and customizable hotkeys.
  • Keyboard / PS / Xbox(XInput) / D-Input display support.

Import / Export

  • Import from .xlsx and .json (template-compatible).
  • Export to .xlsx, .html, and .json for backup or sharing.
  • Supports current context and all-character batch workflows.

Automation (Planned)

  • Auto-calculation and auto-fill support for selected metrics.
  • Automatically change command and buttons based on the selected control mode.

Function-Based Workflows

Frame Data Compare

What It Does

  • Loads frame data by version for the selected character and control mode.
  • Highlights changed cells between two versions and shows exact value deltas in a tooltip.

How To Do It

  1. Select character and mode.
  2. Choose the base version from the Version dropdown.
  3. Click Compare, choose a target version, then apply comparison.
  4. Hover highlighted cells to inspect exact before/after values.

Specific Uses

  • Patch impact review before updating combo assumptions.
  • Verifying whether route stability changed due to startup/recovery/frame-advantage updates.
  • Preparing per-version notes for your own matchup docs.

Combo Entry

What It Does

  • Creates and edits combo rows for each character and control mode.
  • Converts command text into button icons automatically.
  • Supports multiline notes for setup details and route conditions.

How To Do It

  1. Select character and mode from the portrait selector.
  2. Enter route notation in Command (or use quick input/hotkeys).
  3. Fill required condition fields and numeric result fields.
  4. Add more specific setup or other details in Notes where needed.

Specific Uses

  • Building practical match combos with explicit start conditions.
  • Maintaining training-mode tested damage/carry/resource outcomes.
  • Documenting route variants (counter hit, punish counter, CA state, etc.).
  • Notation requirements are summarized in Notations Used.

Table Control (Rows / Columns / Sorting)

What It Does

  • Controls table density and focus using row toggles and column presets.
  • Supports strict duplicate detection and deletion for the current character context.
  • Provides sortable headers/subheaders for quick ordering analysis.

How To Do It

  1. Use Create, Duplicate, Delete, and Dedupe for row operations.
  2. Use FULL, BASIC, SIMPLE, or CUSTOM for column visibility.
  3. Toggle Frame Meter / Buttons / Notes row visibility as needed.
  4. Click any sortable header/subheader to order the dataset.

Specific Uses

  • Use BASIC/SIMPLE for quick reviews, FULL for in depth analysis.
  • Run Dedupe before sharing/exporting to remove exact duplicates.
  • Click the Close button below the table to collapse the lower input section when you need more table workspace.

Search / Advanced Filter

What It Does

  • Search performs quick text/value lookup across standard searchable fields.
  • ADVANCED SEARCH filters by selected fields, flags, and numeric ranges.

How To Do It

  1. Use the main search bar for broad lookup.
  2. Open ADVANCED SEARCH for field-scoped and range-based filtering.
  3. Set exact/min/max values and options, then apply filter.
  4. Reset filters when starting a new review pass.

Specific Uses

  • Find lethal routes by minimum damage threshold and specific conditions.
  • Filter routes by BO/Counter state, resource thresholds, or character restrictions.
  • Target post-combo states for oki planning and consistency checks.
  • Field meanings are documented in Explanation of Fields.

Import / Export

What It Does

  • Imports combo data from .xlsx and .json.
  • Exports in .xlsx, .html, and .json for backup or sharing.
  • Supports current-context export and all-character batch export flows.

How To Do It

  1. Click Import and select a valid template-compatible file.
  2. Click Export, choose format and scope options.
  3. Select whether to export all fields or currently visible fields.
  4. Validate output and run dedupe beforehand when needed.

Specific Uses

  • .json: best for backup/restore and migration.
  • .xlsx: best for spreadsheet workflows and offline review.
  • .html: best for read-only sharing and visual presentation.
  • If import fails, check sheet names/headers against templates. (Templates are stored in the Templates folder)

Data Management / Recovery

What It Does

  • Combo data is stored in browser localStorage by character/mode context.
  • Automatically loads saved context when switching character.

How To Do It

  1. Use normal editing workflow; data is persisted automatically.
  2. Export backups before major cleanup, importing, etc.
  3. If UI becomes inconsistent, run hard refresh (Ctrl+F5).

Specific Uses

  • Recover from accidental edits by importing recent JSON/XLSX backup.
  • Maintain stable snapshots before large imports or dedupe passes.
  • Check PATCH HISTORY when behavior changes between releases.

Command Structure

  • Use ">" between move parts that are cancel extensions.
  • Use ">>" between move parts that has gaps, not cancel extensions, or anything else that are not part of the "combo" like micro walks, set ups.
  • Use "-" for target combos or follow-up options.
  • Use consistent move notation for sorting, filtering, and icon conversion to work correctly.
  • Air routes should include jump notation where required by your current format.

Symbols / Tokens

Icon Notation Meaning Usage Note
1/2/3/4/5/6/7/8/9 Direction (numpad notation) Relative to facing direction.
2(Charge) / 4(Charge) / 360 Charges / 360 (numpad notation) Relative to facing direction.
LP/MP/HP/LK/MK/HK/P/K Classic attack notation Used for attacks in Classic control.
L/M/H/SP/Auto Modern attack notation Use for attacks in Modern control. The tool is planned to distinguishes the use of SP button and full command (for damage calculation, etc.).
Throw/DP/DI/DR/CR Common tokens Common notation tokens. The tool does not recognize regular imputs like LP+LK for throws.
Jump/Hold/Or/Any/[ ] Utility tokens Used for jump state, hold, alternatives, wildcard, and frame-delay or other timing notes.
> / >> / - Separation tokens Used for separating move parts or target combo inputs.

Examples

  • 2LK > 2LP > 236LK (basic grounded route)
  • Jump HK > DR > 2MP > 214MP Hold > SA3 (air starter into super)
  • DI >> 214MP >> HP - HP > 236LP (Drive Impact to set up and to the main combo)
